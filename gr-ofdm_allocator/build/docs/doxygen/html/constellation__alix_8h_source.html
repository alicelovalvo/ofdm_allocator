43  /*!
44  * \brief An abstracted constellation object
45  * \ingroup symbol_coding_blk
46  *
47  * \details
48  * The constellation objects hold the necessary information to pass
49  * around constellation information for modulators and
50  * demodulators. These objects contain the mapping between the bits
51  * and the constellation points used to represent them as well as
52  * methods for slicing the symbol space. Various implementations are
53  * possible for efficiency and ease of use.
54  *
55  * Standard constellations (BPSK, QPSK, QAM, etc) can be inherited
56  * from this class and overloaded to perform optimized slicing and
57  * constellation mappings.

127  /*! \brief Generates the soft decision LUT based on
128  * constellation and symbol map.
129  *
130  * \details Generates the soft decision LUT based on

132  * the noise power in the channel as \p npwr.

134  * \param precision Number of bits of precision on each axis.

137  * This is expensive to compute.
138  */
139  void gen_soft_dec_lut(int precision, float npwr=1.0);

141  /*! \brief Calculate soft decisions for the given \p sample.
142  *
143  * \details Calculate the soft decisions from the given \p sample
144  * at the given noise power \p npwr.
145  *
146  * This is a very costly algorithm (especially for higher order
147  * modulations) and should be used sparingly. It uses the
148  * #gen_soft_dec_lut function to generate the LUT, which
149  * should be done once or if a large change in the noise floor
150  * is detected.
151  *
152  * Instead of using this function, generate the LUT using the
153  * #gen_soft_dec_lut after creating the constellation object
154  * and then use the #soft_decision_maker function to return the
155  * answer from the LUT.
156  *
157  * \param sample The complex sample to get the soft decisions.
158  * \param npwr Estimate of the noise power (if known).
159  */
160  virtual std::vector<float> calc_soft_dec(gr_complex sample, float npwr=1.0);
161 
162  /*! \brief Define a soft decision look-up table.
163  *
164  * \details Define a soft decision look-up table (LUT). Because
165  * soft decisions can be calculated in various ways with various
166  * levels of accuracy and complexity, this function allows
167  * users to create a LUT in their own way.
168  *
169  * Setting the LUT here means that #has_soft_dec_lut will return
170  * true. Decision vectors returned by #soft_decision_maker will
171  * be calculated using this LUT.
172  *
173  * \param soft_dec_lut The soft decision LUT as a vector of
174  * tuples (vectors in C++) of soft decisions. Each
175  * element of the LUT is a vector of k-bit floats (where
176  * there are k bits/sample in the constellation).
177  * \param precision The number of bits of precision used when
178  * generating the LUT.
179  */
180  void set_soft_dec_lut(const std::vector< std::vector<float> > &soft_dec_lut,
181  int precision);
